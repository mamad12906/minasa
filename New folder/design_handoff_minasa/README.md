# تسليم التصميم — مِناصة (minasa)

> **Minasa** — تطبيق تحليلات وإدارة علاقات زبائن للتجارة الإلكترونية (عربي RTL)
> ١١ شاشة موبايل + Dashboard سطح مكتب + نظام تصميم كامل (داكن/فاتح)

---

## 1. نظرة عامة

**مِناصة** تطبيق iOS موجّه للتجار وأصحاب المتاجر الإلكترونية في السوق السعودي/الخليجي. يجمع:

- **نظرة شاملة على الأداء** (مبيعات، إيرادات، طلبات، منتجات)
- **CRM كامل** (قاعدة زبائن، شرائح، ملفات شخصية، دليل)
- **استبصارات ذكية ✦** — تحوّل الأرقام إلى قرارات، بلهجة عربية مباشرة وودودة
- **تنبيهات، تقارير قابلة للتصدير، وإعدادات متجر**

الواجهة كلها **RTL عربية بالكامل**، مع أرقام **عربية-هندية** (٠١٢٣٤٥٦٧٨٩) بخط tabular للحفاظ على المحاذاة في الجداول والرسوم البيانية.

## 2. حول ملفات التصميم في هذه الحزمة

> **مهم جدًا:** الملفات في هذه الحزمة هي **مراجع تصميم (design references) مصنوعة بـ HTML + React + Babel inline** — وهي نماذج أوّلية (prototypes) تُظهر الشكل والسلوك المقصود، **وليست كود إنتاج للنسخ المباشر**.

المطلوب من المطوّر أن **يعيد بناء هذه التصاميم داخل بيئة المشروع الهدف** (React Native / Flutter / SwiftUI / Jetpack Compose / Next.js… حسب تقنية مشروعك)، مستخدمًا الأنماط والمكتبات الموجودة فيه. إذا لم توجد بيئة بعد، اختر الأنسب للمنصّة المستهدفة (للتطبيق الموبايل، **React Native** أو **Flutter** هما الأقرب لهذه المكوّنات البصرية).

**استخدم ملفات HTML للمرجع البصري فقط؛ افتحها في المتصفّح ورجع لها كأنها Figma حيّ.**

### 5.2 شاشات الموبايل (لوحة مختلفة — نعناعي على أسود)

| التوكن | القيمة |
|---|---|

### 5.3 الثيم الفاتح

متوفّر كاملًا في `design-system.css` تحت `[data-theme='light']`. يُفعَّل بإضافة `data-theme="light"` على `<html>`.

### 5.4 الخطوط

```css
font-family: 'IBM Plex Sans Arabic', 'Rubik', system-ui, sans-serif;
```

- **IBM Plex Sans Arabic** — النص العربي (الأوزان: 300, 400, 500, 600, 700)
- **Rubik** — الأرقام (tabular)
- **JetBrains Mono** — الأكواد والقيم الثابتة

**القياسات:**

| العنصر | الحجم | الوزن | line-height |
|---|---|---|---|
| `h1` | 28px | 700 | 1.2 |
| `h2` | 22px | 600 | 1.25 |
| `h3` | 16px | 600 | 1.35 |
| `h4` | 14px | 500 | 1.4 |
| body | 14px | 400 | 1.6 |
| `.eyebrow` | 11px | 600 | uppercase + letter-spacing: 0.08em |
| `.label` | 11px | 500 | — |

### 5.5 الأرقام العربية-الهندية

كل الأرقام في الواجهة **عربية-هندية** (٠١٢٣٤٥٦٧٨٩) وليست غربية. استخدم:

```js
const toArabicDigits = (n) => String(n).replace(/\d/g, d => '٠١٢٣٤٥٦٧٨٩'[d]);
```

الأرقام دائمًا **LTR** داخل سياق RTL للحفاظ على التنسيق. طبّق `direction: ltr; unicode-bidi: embed;` على عناصر الأرقام.

### 5.6 المسافات (Spacing Scale)

## 8. الاستبصارات الذكية ✦ — نمط متكرّر

عنصر تصميم مميّز يظهر في: الرئيسية، الزبائن، التقارير.

**البنية:**
- أيقونة ✦ (زهرة/نجمة رباعية) بلون `--brand` أو `--accent`
- عنوان قصير (14-16px, وزن 600)
- جملة واحدة تفسيرية (13px, `--text-secondary`)
- CTA اختياري (زر صغير)

**نبرة النسخ:** مباشرة، ودودة، عربية فصيحة بسيطة. مثال:
- _"زبوناتك VIP اشترين أقل بـ ٢٣٪ هذا الأسبوع — جرّبي حملة خاصة؟"_
- _"منتج «عباية ملكية» نفد من المخزون، وعليه ١٢ طلب منتظر."_

## 9. التفاعلات والسلوك

### الانتقالات
- جميع الانتقالات بين الشاشات: `transform` + `opacity`، مدة 250ms، easing `cubic-bezier(0.2, 0, 0, 1)`
- Hover على البطاقات: 150ms ease
- Bottom sheets: spring animation

### الحالات
- **Loading:** skeleton screens (ليس spinners) بخلفية `--bg-elevated`
- **Empty:** رسمة بسيطة + عنوان + CTA
- **Error:** chip `danger` + رسالة + إعادة محاولة

### Haptics (موبايل)
- نجاح → light impact
- خطأ → warning notification
- ضغط على CTA رئيسي → selection

## 10. إدارة الحالة

الشاشات المرجعية كلها **static / presentational** — لا state حقيقي فيها. عند التنفيذ:

- **قائمة الزبائن:** state محلي للفلاتر + البحث + الفرز
- **لوحة المعلومات:** data fetching من API (اقتراح: React Query / SWR)
- **التنبيهات:** WebSocket أو polling
- **الإعدادات:** form state مع validation (اقتراح: react-hook-form + zod)

## 11. الاستجابة (Responsive)

- **الشاشات المرجعية مصمّمة على 402×874** (iPhone 14 Pro).
- عند التنفيذ، استخدم **Safe Area insets** على iOS.
- للتابلت: استخدم `minasa-desktop.html` كمرجع.
- الحد الأدنى للعرض المدعوم: **360px**.

## 12. الأصول (Assets)

- **الأيقونات:** مرسومة كـ inline SVG في `components/icons.jsx` — انسخها كملفات SVG مستقلّة أو استخدم مكتبة مثل Lucide (معظمها من هناك).
- **الخطوط:** من Google Fonts (IBM Plex Sans Arabic, Rubik, JetBrains Mono).
- **الصور:** placeholders ملوّنة فقط في التصميم الحالي — **ستحتاج لاستبدالها بصور منتجات حقيقية**.

## 13. الخطوات المقترحة للتنفيذ

1. **ابدأ بـ `tokens/tokens.css`** — انسخها أو حوّلها لنظام التوكنز في مشروعك.
2. **ابن المكوّنات الأساسية أوّلًا** (Button, Card, Chip, Avatar, Input) — هذه تُستخدم في كل شاشة.
3. **نفّذ Dashboard (شاشة 02) أوّلًا** — تحتوي على معظم الأنماط.
4. **ثم شاشات CRM** (06, 07, 08) — الأكثر تعقيدًا.
5. **الشاشات البسيطة أخيرًا** (09, 10, 11).
6. **اختبر RTL بدقة** — خاصّة المحاذاة، الأيقونات الاتجاهية، والأرقام.
